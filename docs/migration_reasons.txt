# System Migration & Architecture Overhaul: Reasoning

## 1. Overview
We are transitioning **TigerResearchBuddy** from a V1 "Prototype" stack to a V2 "Offline Intelligence" stack. This document outlines the technical limitations of the legacy system and the advantages of the new open-source architecture.

## 2. Component Migration

### A. The Scraper (`rit_crawler.py` -> `SmartCrawler`)
*   **Legacy Approach**: `BeautifulSoup` + Regex + CSS Selectors.
*   **Problem**:
    *   **Brittle**: Broke whenever RIT changed a CSS class (e.g., Professor Kinsman's Department/Bio).
    *   **Incomplete**: Missed complex nested data like "Education" or "Research Interests" when formatting varied.
    *   **High Maintenance**: Required constant manual patching (`patch_data.py`).
*   **New Approach**: **LLM-based Extraction (Qwen)**.
    *   **Benefit**: The model "reads" the page like a human. It performs **Semantic Parsing**, ensuring near-100% accuracy even if the HTML structure changes.

### B. The PDF Engine (`chunking` -> `DeepDistiller`)
*   **Legacy Approach**: Extracted raw text and chopped it into 500-word chunks.
*   **Problem**:
    *   **Loss of Context**: Searching for "methodology" returned random fragments.
    *   **Vector Noise**: Resulted in hundreds of low-quality vectors per paper.
*   **New Approach**: **Semantic Distillation (Qwen)**.
    *   **Benefit**: Reads the *entire* paper (Abstract + Intro + Conclusion) to generate a high-density **Research Card**. This ensures the vector index contains the *thesis* of the paper, not just words.

### C. The Database (`ChromaDB` -> `LanceDB`)
*   **Legacy Approach**: `ChromaDB` (File-based).
*   **Problem**:
    *   **Performance**: Slow filtering on metadata.
    *   **Scalability**: Not optimized for large datasets on local disk.
*   **New Approach**: **`LanceDB`**.
    *   **Benefit**: **Serverless & Embedded**. Uses the **Lance** columnar format (Apache 2.0). Optimized for maximum throughput on local SSDs (perfect for Mac). Zero Docker overhead.

### D. The Embeddings (`MiniLM` -> `Nomic`)
*   **Legacy Approach**: `all-MiniLM-L6-v2` (384 dim, 512 context).
*   **Problem**: Truncated useful information from research papers.
*   **New Approach**: **`nomic-embed-text-v1.5`** (768 dim, 8192 context).
*   **Benefit**: Can embed entire research summaries without truncation. Runs efficiently on Mac/Metal via Ollama.

### E. The Intelligence (`Keyword Search` -> `Concept Graph`)
*   **Legacy Approach**: Similarity Search only.
*   **Problem**: Couldn't answer "Who collaborates with Prof X?" or "What is the biggest research cluster?".
*   **New Approach**: **`NetworkX` (Knowledge Graph)**.
    *   **Benefit**: Maps relationships (`Authored`, `Cites`, `Researches`). Enables **GraphRAG** (graph traversal + vector search) for "God Mode" insights.

## 3. Summary
This migration moves the system from a "naive keyword matcher" to a true **Cognitive Research Assistant** that leverages the full power of modern local LLMs and efficient data structures.

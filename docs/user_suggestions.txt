# User Suggestions & System Modernization Plan

## 1. Intelligent Data Ingestion ("Smart Scraper")
**Current State**: `rit_crawler.py` uses BeautifulSoup with hardcoded CSS selectors.
**Problem**: Brittle. Breaks when RIT updates their site layout (as seen with Prof. Kinsman).
**Goal**: Use LLM (Qwen) to parse unstructured HTML into structured JSON.
**Approaches**:
*   **LLM Extraction**: Pipe cleaned HTML/Markdown to Qwen-2.5 with a strict JSON schema prompt to extract: `Name`, `Title`, `Dept`, `Bio`, `Interests`.
*   **Vision-Language Model (VLM)**: Use a VLM to "look" at the profile page screenshot and extract data (more robust to DOM changes).

## 2. Advanced Parsing & Enrichment
**Current State**: `paper_downloader.py` extracts text from first 30 pages. Tags are generated by simple keyword extraction.
**Goal**: Higher quality insights from PDFs.
**Approaches**:
*   **LLM Tagging**: Use Qwen to analyze the Abstract + Intro and generate "Concept Tags" and "Research Pillars" rather than just keywords.
*   **FlashAttention**: Use models supporting FlashAttention for faster processing of long PDF contexts.
*   **Correlation Matrix**: Analyze all faculty papers to build a "Co-Authorship" or "Shared Topic" matrix.

## 3. Storage & Retrieval Upgrades
**Current State**: 
*   **Embedding**: `all-MiniLM-L6-v2` (Fast but older, 384d).
*   **DB**: `ChromaDB` (Local file-based).

**Suggestions**:
*   **Embeddings**: Switch to **`nomic-embed-text-v1.5`** (Matryoshka embeddings, 8192 context window). It's open-source, runs on Ollama, and beats OpenAI Ada-002.
*   **Vector DB**: 
    *   **Qdrant**: High-performance, Rust-based, specialized in filtering (e.g., "Find professors in Dept X with >100 citations").
    *   **LanceDB**: Serverless, highly efficient for local storage, better for larger datasets than Chroma.
*   **Hybrid Search**: Combine Vector Search (Semantics) with Keyword Search (BM25) for precision.

## 4. Advanced "Knowledge Graph" (Concept Mapping)
**Goal**: Make the model "precise and efficient".
**Approaches**:
*   **GraphRAG**: Instead of just chunks, build a Knowledge Graph:
    *   `[Node: Professor] --(RESEARCHES)--> [Node: Topic]`
    *   `[Node: Topic A] --(RELATED_TO)--> [Node: Topic B]`
*   **Association Matrix**: Pre-calculate similarity scores between all professors offline. When a user asks "Who is like Kinsman?", query the matrix O(1) instead of searching.

## 5. Performance Improvements
*   **FlashAttention**: Ensure local LLM backend (Ollama) utilizes FlashAttention for long-context RAG (e.g., reading 5 papers at once).
*   **Quantization**: Use `Q4_K_M` or `Q5_K_M` quants for Qwen to balance speed/accuracy.
